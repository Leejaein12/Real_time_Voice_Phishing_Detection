> 평가 스크립트: `test_scripts/run_mixed_test.py`

## 1. 모델 구조

| 항목 | 내용 |
|------|------|
| 베이스 모델 | KoELECTRA-base-v3 (한국어 ELECTRA) |
| 분류 방식 | 멀티라벨 (Multi-label Classification) |
| 출력 레이블 | `[기관사칭, 금전요구, 개인정보]` — 3개 독립 판단 |
| 손실 함수 | BCEWithLogitsLoss (이진 교차 엔트로피, 멀티라벨용) |
| 추론 임계값 | 각 레이블 Sigmoid ≥ 0.5 시 탐지 |
| 파라미터 수 | 약 110M (hidden 768, attention heads 12, layers 12) |

레이블 예시:
```
[1, 0, 0]  → 기관사칭만
[0, 1, 0]  → 금전요구만
[1, 1, 0]  → 기관사칭 + 금전요구 복합
[0, 0, 0]  → 정상
```

## 4. 탐지 예시

### 4-1. 정상 탐지 (피싱 → 피싱으로 올바르게 판정)

**기관사칭:**
> "여보세요? 저는 검찰 수사관 최진혁이라고 합니다. 지금 중요한 이야기입니다.  
> 당신 명의의 통장이 범죄에 연루되었다고 나왔습니다. 통장 개설한 적 없으시다고요?"  
> → **기관사칭 탐지** ✓

**금전요구:**
> "고객님 안녕하세요. 대출 관련해서 꼭 말씀드릴 게 있어서 연락 드렸습니다.  
> 금리가 현재 많이 낮아서 가능하면 빨리 진행하셔야 해요."  
> → **금전요구 탐지** ✓

**개인정보:**
> "고객님, 지금 카드 인증 작업 중입니다. 카드 번호와 만료일을 확인해 주시면 감사하겠습니다.  
> 이렇게 해야만 처리가 가능합니다."  
> → **개인정보 탐지** ✓

**복합 (기관사칭 + 금전요구):**
> "안녕하세요, 서울지검의 최강민 검사입니다. 이번 사건과 관련해서는 제3자에게 이야기하지 말아 주세요.  
> 저희는 자산 보호를 위해 몇 가지 절차를 진행할 텐데, 동의하실 건가요?"  
> → **기관사칭 + 금전요구 복합 탐지** ✓

**v4 테스트 오탐 1건 (실제 결과):**

|-------:|---------------|--------|
| **99.4%** | 개인정보 | "네, 고객님, 성함과 주민번호 앞자리를 말씀해 주시면 합격자 명단에서 확인 후 안내 도와 드리겠습니다." |

### 6-1. 발견된 미탐(FN) 사례 — 탐문형 피싱

`mixed_test_03.json` (KBS 공개 시나리오 원본 포함) 테스트에서 v4 모델이 탐지하지 못한 문장:

> "본인이 개설한 적이 없다는 말씀이시죠? 그럼 혹시 예전에 제3자에게 본인 명의로 빌려줬다던가 신분증이나 운전면허증 분실한 적은 있으신가요?"  
> — KBS 공개 보이스피싱 시나리오 (2023.03), 검1선 8번 발화

| 항목 | 값 |
|------|----|
| 키워드 점수 | 60점 (제3자·신분증·명의 등 키워드 4개 매칭) |
| KoELECTRA 실행 | ○ (키워드 필터 통과) |
| KoELECTRA 판정 | **정상** (위험도 < 50%) → **미탐** |
| 실제 라벨 | `[1, 0, 1]` 기관사칭 + 개인정보 |

**미탐 원인 분석:**  
이 발화는 경찰청 사무관을 사칭한 사기범이 피해자에게 사건 연루 가능성을 탐문하는 단계로, 금전이나 개인정보를 **직접 요구하지 않는다.** 학습 데이터가 "주민번호 알려주세요", "계좌로 이체하세요" 같은 직접 요구형에 치우쳐 있어, 질문·공감·상황 확인으로 신뢰를 쌓는 **탐문형 수법**을 정상 통화로 오인하였다.

### 6-2. v5 보완 학습 계획

**탐문형 피싱 데이터 500개 추가 증강 후 재학습 (`augment_v5.py`)**

| 항목 | 내용 |
|------|------|
| 시드 ① | KBS_001 검1선 전체 대화 (탐문형 패턴이 가장 밀집된 시나리오) |
| 시드 ② | 실제 보이스피싱 통화 전사(transcripts.json) 중 탐문형 키워드 포함 80개 |
| 증강 라벨 조합 | `[1,0,0]` 150개 / `[1,0,1]` 150개 / `[1,1,1]` 200개 |
| 증강 방식 | GPT-4o-mini — 탐문·확인·공감 어조 명시한 프롬프트 |
| 출력 | `data/v5/` (v4 1,510개 + 신규 500개 = 2,010개) |

탐문형 프롬프트 핵심 지시:
```
직접 금전/정보를 요구하는 대신
사실 확인 질문·상황 탐문·공감 유도로 신뢰를 쌓는 단계의 발화를 생성
예: "혹시 신분증 분실한 적은 있으신가요?",
    "본인이 개설한 적이 없다는 말씀이시죠?",
    "지금 사용 중이신 금융권이 어떻게 되십니까?"
```

**개선 계획:**
1. `augment_v5.py` 실행 → `data/v5/` 탐문형 보강 데이터 생성
2. `train_colab.py` `DATA_VERSION = 'v5'` 로 재학습
3. `mixed_test_03.json` 재평가 → 탐문형 FN 감소 확인
4. 뉴스/보도 정상 샘플 추가 및 Android TFLite int8 양자화
